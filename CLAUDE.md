**プロジェクト名**: OsmDotRoute（確定 2026-05-18）

## 段階的開発計画

### Phase 0: 要件定義（現在）
- 要件定義書を作成・確定する
- 大まかな API デザインの方向性を決める
- ライセンスとリポジトリ公開戦略を決める

### Phase 1: 経路探索エンジン独自化
- 動的制約対応の Dijkstra ベース経路探索を実装
- グラフ構造は Itinero の `RouterDb.Network` をそのまま借用
- OSM PBF パース・RouterDb 永続化は Itinero に任せる
- 親プロジェクト（災害廃棄物処理シミュレーション）から呼べる NuGet パッケージとして提供

### Phase 2: 中間グラフフォーマット
- 独自バイナリ形式のグラフを設計・実装
- Itinero RouterDb → 独自形式への変換ツール作成
- ランタイムから Itinero 依存を削除
- この時点で本ライブラリ単体で配布可能になる

### Phase 3: OSM PBF パーサー独自化
- OSM PBF を直接読み込む独自パーサーを実装
- protobuf-net または独自プロトコルバッファ実装
- way / node / relation の解釈、車道判定ロジック
- Itinero への依存を完全に排除
- **着手判断は Phase 1/2 完了後、市場・技術環境を見て判断する**

本プロジェクトの**最初の顧客**は親プロジェクト `災害廃棄物処理シミュレーション` です。
