## 1. 目的とゴール

**目的**: 親プロジェクト「災害廃棄物処理シミュレーション」が Itinero 1.5.1 から OsmDotRoute Phase 3 へ移行するための**マイグレーションガイド**を作成する。本ステップでは親プロジェクトのコードを直接変更せず、ドキュメントのみを成果物とする。

**方針変更の経緯**: v0.1 では本プロジェクト主導で親プロを直接修正する計画だったが、ユーザー判断（2026-05-28）により「ガイド作成 → 親プロ側で実施」の分離運用に切り替え。

**Done 判定**:

1. マイグレーションガイドが本プロジェクト `Documents/migration_from_itinero.md` に作成される ✅
2. 同ガイドが親プロジェクト `Documents/` にもコピー配置される ✅
3. ガイドに以下が含まれる: ✅
   - Itinero → OsmDotRoute API 対応表
   - 親プロ各ファイルの具体的 before/after コード例
   - Extractor 子プロセス呼出パターン
   - `.routerdb` → `.odrg` 移行手順
   - ProjectReference 設定例
   - 既知の差分と注意事項
4. **親プロ側で移行作業を実施**し、トラブル / 不足機能があれば OsmDotRoute 側で対応
5. 親プロ移行完了後に設計書 `phase3_design.md` §8 を肉付け
6. OsmDotRoute 側 `dotnet test` **672 件 pass** を維持

**完了判定タイミング**: 親プロ側での移行作業完了を待ってから 3F 正式完了とする（ユーザー判断 2026-05-28）。トラブルや不足機能の発見に備え、3F は「ガイド作成済 → 親プロ移行待ち」状態で一時保留する。
