## 1. 前提

### 1.1 検証範囲

- **対象**: Phase 1 機能要件と性能要件（市単位）
- **除外**: 親プロジェクト統合実証（ステップ 16）— **Phase 3 完了まで延期**（2026-05-20、ユーザー判断）
- **除外**: 都道府県単位 RouterDb での REQ-NFR-001〜003 最終確認 — Phase 3 完了後に再検証

### 1.2 既に検証済の項目（再検証不要）

- 単体テスト 153/153 通過（リポジトリの xUnit テスト）
- ベンチマーク（津島市 57k エッジで全要件達成）
- ステップ 13〜14 で MapVerifier 1.0.0 リリース時に end-to-end 動作確認済（要件定義書 v1.9 で `[x]` マーク済の機能要件 全件）

### 1.3 本チェックリストの位置付け

**Phase 1 完了確定のための最終ユーザー確認**。各項目を MapVerifier 上で実行し、想定動作と一致することを確認する。差分があればコメント記録 → Phase 1 確定可否を判断。

## 2. 起動手順

```powershell
# リポジトリルートで実行
./start-map-verifier.ps1
```

ブラウザで http://localhost:5173 が自動で開く。Server (port 5279) と Web (port 5173) が別ウィンドウで起動する。

## 3. 検証チェックリスト

### 3.1 RouterDb 読込（REQ-MAP-001〜002）

- [X] **RD-1**: 「マップ範囲設定」で `default.routerdb`（または任意の RouterDb）を指定して読込
- [X] **RD-2**: 統計表示で頂点数・辺数・bbox が画面上に出ること
- [X] **RD-3**: 道路ネットワーク（LineString）が地図に描画されること

### 3.2 経路計算（REQ-RTE-001〜008）

- [X] **RT-1**: 起点・終点を地図クリックで指定し、car プロファイルで経路が描画される
- [X] **RT-2**: 同じ起終点で pedestrian プロファイルに切替 → 経路が変わる（または同一でも警告なし）
- [X] **RT-3**: 道路ネットワーク外の点を指定したとき、エラー表示またはスナップ失敗が明示される
- [X] **RT-4**: 距離・所要時間が UI に表示される

### 3.3 メッシュ表示（REQ-RST-016〜018）

- [X] **MS-1**: 1km / 500m / 250m メッシュをトグル表示できる
- [X] **MS-2**: 各メッシュコードが地図上で確認できる（クリックでコード表示など）

### 3.4 動的制約：進入不可エリア（REQ-RST-001〜003）

- [X] **BL-1**: マウスドラッグでポリゴンを描画 → BlockArea として登録
- [X] **BL-2**: メッシュ指定モードでメッシュをクリック → BlockArea として登録
- [X] **BL-3**: 登録した BlockArea が地図上に色分けで表示される（赤系統）
- [X] **BL-4**: BlockArea を経路上に置いて再計算 → 迂回経路または「経路なし」結果

### 3.5 動的制約：難所エリア（REQ-RST-004〜007、REQ-PRF-007〜014）

- [X] **DF-1**: 難所タイプ `flood` でポリゴン登録 → 経路はそのまま通過するが速度が低下する
- [X] **DF-2**: 難所タイプ `debris` で登録 → flood と挙動が異なる（速度低下率や通行可否）
- [X] **DF-3**: `narrow` / `damaged` / `closed` で同様に挙動差を確認
- [X] **DF-4**: 同一エリアに複数の難所タイプを重ねた場合 → speedFactor が積算される（REQ-RST-030〜032）

### 3.6 制約管理（REQ-RST-008〜012）

- [X] **MG-1**: 制約一覧表示（登録順または ID 順）
- [X] **MG-2**: 個別 ID で削除 → 経路再計算で影響消える
- [X] **MG-3**: タグ指定で一括削除（REQ-RST-010）
- [X] **MG-4**: 全クリア（REQ-RST-009）
- [X] **MG-5**: 制約変更後に経路再計算 → **即時反映される**（REQ-RST-012）

### 3.7 GML 入力（REQ-RST-020〜028, REQ-RST-040）

- [X] **GM-1**: 国土数値情報 A31 (`D:/ハザードデータ/A31-12_24_GML/`) のような KSJ GML 3.2 ファイルをアップロード
- [X] **GM-2**: 範囲指定なしで全フィーチャ採用
- [X] **GM-3**: 範囲指定ありで bbox 内のフィーチャのみ採用される（REQ-RST-040）
- [X] **GM-4**: 不正な GML（XML ではない / xlink 解決失敗）で適切なエラー表示
- [X] **GM-5**: 採用されたポリゴンが地図上に描画される

### 3.8 性能体感（参考、定量はベンチで実施済）

- [X] **PF-1**: 経路計算が 100ms 程度の体感速度で完了
- [X] **PF-2**: 制約 50 件登録後でも経路計算が体感で遅くならない
- [X] **PF-3**: 計画書 §7.4 / ベンチ結果 §7 — DifficultyArea を経路直線上に意図的に配置したとき、Dijkstra 探索拡大による顕著な遅延がないこと（観測あれば本チェックリストにコメント記録、設計書 §18「制約事項と既知の課題」に申し送り）

### 3.9 操作性・UI

- [X] **UI-1**: 地図のズーム・パンが滑らかに動く
- [X] **UI-2**: エラーや警告が UI 上で確認可能
- [X] **UI-3**: シナリオを切替えても状態がリークしない
